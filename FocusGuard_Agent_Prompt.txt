Context & GoalYou are building FocusGuard, a Chrome browser extension MVP for a hackathon. It tracks student attention and cognitive state across browser tabs in real time, classifies whether their focus is productive or misdirected, accumulates a cognitive debt score over time, and delivers contextual AI-powered nudges via the Gemini API.The system has two parts that you must build: a Chrome extension (JavaScript / MV3) and a local Python backend (FastAPI). They communicate over a local HTTP API.ℹ  Build PhilosophyBuild every file completely. No // TODO, no pass without explanation, no placeholder functions.Every file must be independently importable/runnable without errors even when drop-in modules are absent.The JS extension must never crash if the Python server is offline — graceful degradation always.The Python server must never crash if any drop-in ML module is absent — every such import is wrapped in try/except.Comment every non-obvious logic block. Judges will read this code.All timestamps are unix milliseconds throughout the entire system.Do not use any external JS libraries in the extension except Chart.js (in popup only).🚫  Critical Exclusions — Do NOT Build Theseinference/       — ML inference wrappers will be provided separately by the ML team.models/          — All .pkl files and loader.py will be provided separately.features/        — All signal processing and sequence building will be provided separately.popup/           — The popup dashboard UI will be built separately.Where these modules are imported, write the import statement and add a comment:  # ⚠️ DROP-IN MODULE — provided separately by ML team  # Expected interface: <describe the function/class and its return shape>Part 1 — Chrome Extension (JavaScript, MV3)1.1  File StructureBuild all files listed below.
